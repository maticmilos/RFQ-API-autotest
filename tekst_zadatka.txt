NEGATIVE BUGS - API errors, failures, bad behavior
Bug N1: Empty Text Returns 200 Instead of 400
Title: Empty text input incorrectly returns 200 OK instead of an error
Environment: Development, Postman v11.0, Windows 11
Severity: Medium
Priority: High
Steps to Reproduce:
1. Open Postman, set the Authorization header to ApiKey {{apiKey}}
2. Send a POST request to https://intelligence-dev.setvi.com/api/rfq/upload-free-
text
3. Set the body to:
json
{
"text": "",
"topK": 3,
"threshold": 0.5,
"enablePrivateLabelRanking": false
}
4. Send request
Actual Result:
- Status: 200 OK
- Response body: {"statusId": "abc123", "matchedProducts": []}
- No error message
Expected Result:
- Status: 400 Bad Request
- Response body should contain an error like: {"error": "Validation failed", "message":
"Text cannot be empty"}

Title: Invalid URL format triggers unhandled server exception (500)
Environment: Development, Postman v11.0
Severity: High
Priority: High
Steps to Reproduce:

2. Set the body to:

- Response: {"error": "Invalid URL", "message": "Please provide a valid URL starting
with http:// or https://"}
Bug N3: Threshold Parameter Ignored - Low Scores Included
Title: threshold=0.8 still returns products with 0.4 similarity
Environment: Development, Postman v11.0
Severity: High
Priority: Medium
Steps to Reproduce:
1. Send POST to /upload-free-text with body:
json
{
"text": "Cutting Board",
"topK": 5,
"threshold": 0.8,
"enablePrivateLabelRanking": false
}
2. Check response
Actual Result:
- Returns products with similarity scores: 0.85, 0.72, 0.61, 0.45, 0.38
- Products with scores < 0.8 are not filtered out
Expected Result:
- Only return products with similarityScore ≥ 0.8
- Array length should be ≤ 2 (only 0.85 and maybe 0.72)
Bug N4: topK Parameter Not Respected - Always Returns 5
Title: topK parameter ignored, always returns exactly 5 results.
Environment: Development
Severity: Medium
Priority: Medium

2. Test with topK=10: {"text": "container", "topK": 10, "threshold": 0.5}
3. Count results
Actual Result:
- topK=1 → returns 5 products
- topK=10 → returns 5 products

Bug P1: Private Label Ranking Flag Has No Effect
Title: Private Label Ranking doesn't change result order
Environment: Development
Severity: Medium
Priority: Medium
Steps to Reproduce:
1. Send request with flag false: {"text": "cutting board", "enablePrivateLabelRanking":
false}
2. Send request with flag true: {"text": "cutting board", "enablePrivateLabelRanking":
true}
3. Compare results
Actual Result:
- Both requests return identical product lists in the same order
- Private label products (store brands) are not prioritized when flag=true
Expected Result:
- When flag=true, private label products should appear first
- Example: "StoreBrand Cutting Board" should rank above "PremiumBrand Cutting
Board."
Bug P2: High Similarity Score for Irrelevant Product
Title: Unrelated product gets 0.89 similarity score for "Cutting Board"
Environment: Development
Severity: High
Priority: High
Steps to Reproduce:
1. Send: {"text": "Plastic Cutting Board 24x18", "topK": 3, "threshold": 0.5}
2. Check the first result
Actual Result:

2. Run Tests: Click the collection → "Run."

Variables
- `baseUrl`: https://intelligence-dev.setvi.com
- `apiKey`: Your API key (already set)
- `threshold`: 0.5 (default confidence)
Running All Tests
1. Click the collection name "Setvi-Task."
2. Click the "Run" button
3. View results in Runner tab
Troubleshooting
- 401 Error: Check API key in Variables
- 403 Error: URL is blocked - use a different site
- 404 Error: Check the baseUrl is correct
- Slow response: API or network issue
